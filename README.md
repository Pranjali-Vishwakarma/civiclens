# CivicLens — Report. Track. Resolve.

An AI-powered civic issue reporting and smart city accountability platform. CivicLens empowers neighborhoods to flag infrastructure issues, track resolutions in real-time, verify reports, and inspect ward-by-ward civic health analytics.

---

## 🚀 Key Features

1. **AI-Powered Reporting**: 
   - Upload infrastructure photos to get automated category, description, and severity analysis via Google Gemini 1.5 Flash.
   - Mobile camera integration (`capture="environment"`) with fail-safe geolocation fallback to Pune coordinates.
   - Built-in duplicate detection using PostGIS spatial search (within 50 meters of existing reports).

2. **Interactive Live Map**:
   - Live coordinates plotting of reported issues, color-coded by category (Potholes, Streetlights, Water leaks, Garbage, Road damage).
   - Real-time page updates using Supabase realtime subscriptions.
   - Interactive popups showing details, photo uploads, and neighborhood verification logs.

3. **Accountability Analytics Dashboard**:
   - Comprehensive performance metrics: Total Reports, Open Issues, Resolution Rate, and Average Resolution Time.
   - Visual charts (Recharts): Category counts and status breakdowns.
   - **Civic Health Scores**: Ward-by-ward indexes calculated dynamically.
   - **AI Predictive Insights**: Automated weekly hotspot analysis and actionable risk ratings generated by Gemini.
   - **CivicHero Leaderboard**: Community point system highlighting top users.

4. **Community Verification System**:
   - Neighbors can confirm or dispute reports.
   - Automatic status transition to *Community Verified* once threshold verifications are met.

---

## 🛠 Tech Stack

- **Framework**: Next.js (App Router), Tailwind CSS (Theme Configuration), TypeScript
- **AI Model**: Google Gemini 1.5 Flash (via `@google/genai`)
- **Database & Realtime**: Supabase (PostgreSQL with PostGIS extensions)
- **Charts**: Recharts
- **Mapping**: React-Leaflet & OpenStreetMap (custom DivIcons)

## 📦 Getting Started

### 1. Install Dependencies
```bash
npm install
```

### 2. Configure Environment Variables
Create a `.env.local` file in the root directory:
```env
NEXT_PUBLIC_SUPABASE_URL=your-supabase-project-url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-public-key
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key
GEMINI_API_KEY=your-google-gemini-key
```

### 3. Seed Database
Run the standalone database seeder to populate realistic Pune metrics (45 issues, verification logs, and leaderboard points):
```bash
npx ts-node scripts/seed.ts
```

### 4. Run Development Server
```bash
npm run dev
```
Open [http://localhost:3000](http://localhost:3000) to view the application.
